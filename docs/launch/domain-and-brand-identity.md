> **`flipper.com`** is taken (a hardware/IoT company). **`flipper.io`** is also taken.
> **`flipper.ai`** is taken (someone is parking it; a transfer would likely cost $5,000-50,000+).

### My ranking (with reasoning)

1. **`flipper.app`** — shortest, cleanest, makes the brand "Flipper" not "Flipperai". `.app` enforces HTTPS by default. Cheapest tier. *If available, take it.*
2. **`flipperai.app`** — second-best. Keeps the AI association in the name. Cheap.
3. **`getflipper.ai`** — best of the `.ai` options because "get" implies action. Expensive recurring cost (~$70/yr) but signals "this is an AI tool" without explanation.
4. **`useflipper.ai`** — same idea as `getflipper.ai`, slightly less natural in conversation.
5. **`flipperai.com`** — *not yet considered above*. Almost certainly taken or expensive on the secondary market, but worth checking — a `.com` is the gold standard if available at sane price.
6. **`flipthat.ai`** — clever but harder to remember/spell.
7. **`tryflipper.com`** — cheap and clear, but "try" can feel beta-y forever.
8. **`flipper.tools`** / **`flipscout.ai`** — fallback options.

> **My recommendation:** Check `flipper.app`, `flipperai.com`, and `flipperai.app` in that order. Take the first one available. If all three are gone, escalate to `getflipper.ai`.

## Social handle reservation — check before you buy the domain

Once you pick a domain, you want **the same handle on every platform**. Check all of these *before* registering the domain — if `flipperai` is taken on Twitter/X, you might prefer a different domain that has clean handle availability.

For each candidate username (`flipper`, `flipperai`, `getflipper`, `tryflipper`, `flipthat`, `useflipper`, `flipscout`), check:

| Platform           | Direct check URL                                              |
| ------------------ | ------------------------------------------------------------- |
| Twitter / X        | `https://x.com/<handle>` (404 = available, page = taken)      |
| Instagram          | `https://www.instagram.com/<handle>/`                          |
| TikTok             | `https://www.tiktok.com/@<handle>`                            |
| Reddit             | `https://www.reddit.com/user/<handle>`                        |
| GitHub             | `https://github.com/<handle>` (404 = available)               |
| LinkedIn (company) | https://www.linkedin.com/company/setup/new/                   |
| Product Hunt       | `https://www.producthunt.com/@<handle>`                       |
| YouTube            | `https://www.youtube.com/@<handle>`                           |
| Facebook page      | `https://www.facebook.com/<handle>`                           |
| Pinterest          | `https://www.pinterest.com/<handle>/`                         |

**Bulk-check tool:** https://namecheckly.com/ (free, no signup, checks ~12 platforms in one query)

## After you buy the domain

1. **Set up DNS in Cloudflare** (even if you bought elsewhere, transfer to Cloudflare for free DNS).
2. **Add as custom domain in Firebase Hosting**:
   - Console: https://console.firebase.google.com/project/axovia-flipper/hosting/sites
   - Click "Add custom domain"
   - Verify via TXT record (instructions appear in Firebase)
   - Add A records pointed at Firebase's IPs (also shown in console)
   - SSL cert provisions automatically within 24 hours
3. **Update `APP_URL` in GCP Secret Manager** to the new domain.
4. **Update `ALLOWED_ORIGINS`** to include the new domain (and any www / non-www variants).
5. **Update OAuth redirect URIs**:
   - Firebase Auth: console → Authentication → Sign-in method → Google → Authorized domains
   - Google Cloud Console (for OAuth client): https://console.cloud.google.com/apis/credentials → your OAuth 2.0 Client ID → "Authorized redirect URIs"
   - GitHub: https://github.com/settings/developers → your OAuth App → Authorization callback URL
   - Facebook: https://developers.facebook.com/apps/ → your app → Settings → Basic + Facebook Login settings
6. **Update Stripe** webhook endpoint URL: https://dashboard.stripe.com/webhooks → your endpoint → "Update details"
7. **Set up email forwarding** for `support@<domain>`, `hello@<domain>`, `stephen@<domain>` → your inbox.
8. **Set up Resend domain verification** so transactional email comes from `<domain>` not the Resend default: https://resend.com/domains
9. **Update your Twitter/X bio** with the new domain.
10. **Update your GitHub repo description** with the new domain.

> Each of these is a 5-minute task individually, but the cumulative time is ~2 hours. Block out a single session.
